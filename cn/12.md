## 12.1. 又是螺丝刀

![](/images/dongge-apple/12/12-02.webp)

已经想不起来把 “螺丝刀” 当过多少次例子使用了…… 好像还得再来一次。

九十年代末的时候，我有过一把当时售价 515 元的螺丝刀 —— 就是很贵很贵的那种。我用它组装计算机 —— 我开过电脑公司 —— 那时候，拿来一堆配件，把它们组装成一台计算机，只需要一个工具，就是螺丝刀，刀头十字花且有磁力的那种。

那可是我赖以生计的工具之一，所以当然对它的要求格外严格。刀头必须是上等的精钢，十字花头绝对不能用一段时间就被拧坏。关键在于，刀尖的磁力不能太大，必须一次只能吸上一个螺丝钉，因为一下子能吸上太多螺丝钉，每次都要用另外一只手把其它的螺丝钉摘掉；磁力也不能太小，吸在刀尖上的螺丝钉绝对不能轻易脱落，否则有可能造成电脑主板短路甚至烧毁。所以，选择螺丝刀的时候，得带着若干装机专用的螺丝钉逐一测试，综合考虑。

别人都是随便抓来一个螺丝刀就用，我不行，做不到。不是我格外 “事儿妈”，而是 “在想清楚想明白之后如何随便糊弄自己呢？” 当然，认真对待工具，是从小就养成的习惯。

最夸张的时候，我曾经用那把螺丝刀，在 12 个小时之内 —— 就是晚上 6 点钟开始，到第二天清晨 6 点钟，总计组装了 65 台计算机 —— 熟练到组装一台机器只用 10 分钟左右…… 并且，在这个过程中，还要同步穿插着克隆 65 块硬盘，然后还要每一台都要点亮测试通过…… 

那时候我常常开玩笑说，大学里学了四年会计，结果屁也不会；到最后竟然得靠一把螺丝刀养家糊口。其实，我自己知道的，那不是螺丝刀的功劳，也不是螺丝刀很贵才可以的事儿。甚至，也不是懂 DIY 组装电脑的事儿 —— 组装个电脑肯定完全不需要学历吧？

真正起作用的，其实是一种狠劲、韧劲，能专注 12 小时，不睡觉也得把活干完的劲儿。当然，能做到 12 小时之内干完，肯定是因为曾经对整个流程反复琢磨，有经验、有总结，有诀窍，所以才能行云流水地干活，并且还不觉得累，反倒是乐在其中 —— 毕竟，有啥事儿比动手赚钱更开心的呢？尤其，只不过是用一把螺丝刀而已…… 至于买那把螺丝刀的 515 元成本花销，早就赚回不知道多少倍了吧？

## 12.2. 有用与否在于人

![](/images/dongge-apple/12/12-03.webp)

无论什么工具，都肯定是有用的 —— 它们就是为了某种或某些用处被发明被设计的，不是吗？可是，任何工具的作用，其实跟使用者有更大的关系，因为：

> 工具的功效和工具使用者的能力绝对正相关。
>
> A tool is as useful as the user is capable.

我们曾说，计算机是这个时代里最重要的 “平权工具”。计算机其实也是最 “因人而异” 的工具。人们用着各方面都差不多的计算机，可能是相同品牌、相同价格、相同硬件配置、相同系统设置选项的东西，却做着不一样，千差万别，甚至天壤之别的事情。

重点在于 “如何使用工具” 吗？我们反复强调并整理补充的 “通用工具方法论” 当然重要，然而，更重要的显然是使用工具的人。当人人手中都有相同工具的时候，这一点不仅格外重要，甚至是唯一个区别：你到底是谁？你是什么样的人？你本身的能力等级到了什么程度？

这些才是真正的关键。研究工具，完善工具方法论，当然必须且意义非凡，但，“系统的通用工具方法论” 中，必须加上最重要的一条：

> 既然工具为我所用，那么我不行它也不行……

所以，必须从各个角度打造自己，就好像芒格所说的一样，无论想要得到什么，最简单最有效的方法就是 “让自己配得上它”。

这没什么 “难度”。可问题在于，如果我没有通过 “阅读” 去看过 `SVG` 关于文件格式的相关文档，甚至压根不知道 `SVG` 这个概念的存在，我无论如何都没办法跟人工智能说那样的话，提出那样的要求。

在第 11 章里，那张由一根螺旋线和一根随机线条构成的示意图，对 ChatGPT 说的话，其实也是针对 `SVG` 的描述，不过，你还得起码知道另外一个概念，那种螺旋曲线的名称是 “阿基米德螺线”（Archimedean spiral）；而另外一个线条，只不过是 “从同一个点向右上角延伸的随机弯曲线条” 而已，不存在 “难以描述” 的情况。

—— 这个提示词极大概率并不能得到你想要的结果。我还得知道完成这个任务大抵上应该通过调用 Javascript 才能动态生成两条曲线，最终不是一个 `SVG` 文件，而是一个 `HTML` 文件，用浏览器打开才能看到它渲染出来的图形…… 于是，你脑子里得有若干个很多人可能没有的概念及其相互之间的关联…… 所以，我的提示词是：

> Create an *HTML* file that displays an *SVG* containing a *Archimedean spiral* and a curved line. The spiral should be drawn from the center point, expanding clockwise, and have arrows at the end of the path. The curved line should be randomly bent and also have an arrow at the end. Both shapes should be defined using *JavaScript* functions to create the *SVG paths*. The spiral should have a black arrow, and the curved line should have a blue arrow. Please also include relevant *styles* and JavaScript to achieve this.
>
> ---
>
> 创建一个包含**阿基米德螺旋线**和一条随机弯曲的线条的 **SVG** 图像的 **HTML** 文件。螺旋线应从中心点开始，顺时针扩展，并且路径的末端带有箭头。另外一个线条应随机弯曲，并且末端也带有箭头。两种形状的**路径**都应该通过 **JavaScript 函数**生成。螺旋线末端的箭头应该是黑色的，曲线末端的箭头应该是蓝色的。请包含相关的**样式**和 JavaScript 代码以实现此效果。
>
> https://chatgpt.com/share/670b8525-ed54-8009-abea-84617b24ed61

即便如此，获得的结果也不一定直接令人满意。然而，我对这些概念并不陌生，所以，虽然也需要修修补补，反复几次，但总体上，我还是很快就可以通过调整得到自己想要的结果 —— 前后总计也就十分钟的事儿：

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Spiral SVG with Arrows</title>
    <style>
        /*svg {
            border: 1px solid black;
        }*/
    </style>
</head>
<body>
    <svg id="spiral" width="800" height="800" xmlns="http://www.w3.org/2000/svg">
        <defs>
            <marker id="arrow-black" markerWidth="10" markerHeight="10" refX="5" refY="5"
                    orient="auto-start-reverse" markerUnits="strokeWidth">
                <path d="M 0 0 L 10 5 L 0 10 z" fill="black"/>
            </marker>
            <marker id="arrow-blue" markerWidth="5" markerHeight="5" refX="2.5" refY="2.5"
                    orient="auto-start-reverse" markerUnits="strokeWidth">
                <path d="M 0 0 L 5 2.5 L 0 5 z" fill="blue"/>
            </marker>
        </defs>
    </svg>
    <script>
        function createSpiralPath(cx, cy, turns, spacing) {
            let pathData = `M ${cx},${cy}`;
            const points = 100; // Number of points per turn

            for (let i = 0; i < turns * points; i++) {
                const angle = i * (2 * Math.PI / points); // Positive angle for clockwise
                const radius = spacing * angle / (2 * Math.PI);
                const x = cx + radius * Math.cos(angle);
                const y = cy + radius * Math.sin(angle);
                pathData += ` L ${x},${y}`;
            }

            return pathData;
        }

        function createCurvedLine(cx, cy, length) {
            // Create a random curved line path
            const xEnd = cx + length;
            let pathData = `M ${cx},${cy}`;
            
            for (let i = 0; i <= length; i += length / 10) {
                const x = cx + i;
                const y = cy - i + (Math.random() - 0.5) * 50; // Create a random offset for y, extending upwards
                pathData += ` L ${x},${y}`;
            }

            return pathData;
        }

        const svg = document.getElementById('spiral');
        const spiralPathData = createSpiralPath(400, 400, 20, 10);
        const curvedLinePathData = createCurvedLine(400, 400, 200); // Shorter length for the curved line

        const spiralPath = document.createElementNS('http://www.w3.org/2000/svg', 'path');
        spiralPath.setAttribute('d', spiralPathData);
        spiralPath.setAttribute('stroke', 'black');
        spiralPath.setAttribute('fill', 'none');
        spiralPath.setAttribute('marker-end', 'url(#arrow-black)'); // Add black arrow to the end

        const curvedLinePath = document.createElementNS('http://www.w3.org/2000/svg', 'path');
        curvedLinePath.setAttribute('d', curvedLinePathData);
        curvedLinePath.setAttribute('stroke', 'blue');
        curvedLinePath.setAttribute('fill', 'none');
        curvedLinePath.setAttribute('stroke-width', '2'); // Set the line to be thinner
        curvedLinePath.setAttribute('marker-end', 'url(#arrow-blue)'); // Add blue arrow to the end

        svg.appendChild(spiralPath);
        svg.appendChild(curvedLinePath);
    </script>
</body>
</html>
```

其实也没多难，只不过是懂一点编程而已 —— 当然，这也是学一点编程的一点点好处。有了 ChatGPT 之后，我产生了极为强烈的 “系统学习软件工程” 的欲望，并逐步采取行动。反正，学什么都有用，做什么都要学，学什么都要练，练什么都是练自己的脑子……

所以，当然应该卷工具，但更应该卷自己。

## 12.3. 这本书想要改变的是你这个人

![](/images/dongge-apple/12/12-04.webp)

所以，在整本书的每一章里，我所讲解的内容，其核心全都一样，不在于或者不仅在于讨论 ChatGPT 这个工具的使用技巧，更重要的，更多地，我的重点在于同一件事情：

> 把你变成 “**使用同一种工具的另外一种人**”。

人不一样了，工具就不一样。

散落在本书各处的，都是改变你这个人的重点。若是不嫌啰嗦， “结构化” 地罗列一下的话，是这么一个列表：

人工智能这个东西，特别像关于亚瑟王的传说里的，那把插在石头里的宝剑…… 无数人前去尝试，拼尽全力想要拔出宝剑，却都失败了，而年轻的亚瑟路过，轻轻一拉，就把宝剑从石头里拔出来了。关键在于人，若你不是亚瑟，那宝剑对你来说就是插在石头里的一个物件，拔不出，用不上。

同样的道理，你得把你自己变成能够用得上人工智能这个强大工具的人。2024 年夏天的时候，民众在嘲笑各种人工智能 “搞不明白到底 9.11 和 9.9 哪个数字更大”…… 到了秋天，2024 年的诺贝尔奖获得者中有四位人工智能科学家获奖。

其中，诺贝尔化学奖格外受人瞩目，因为化学家戴维·贝克（David Baker）与另外两位人工智能科学家德米斯·哈萨比斯（Demis Hassabis）和约翰·江珀（John Jumper）合作开发出的 AlphaFold2 模型而获奖，这个模型能够快速准确地预测蛋白质的三维结构，解决了一个存在 50 年的科学难题，这一成就极大地推动了生物医学研究和药物开发。

![](/images/nobel_prize_in_chemist.png)

如果住在传说里，他们就是能把宝剑轻松拔出来，而后挥舞着那宝剑去征服大陆的人…… 不是吗？反正，同样的东西，在不同的人手中，就是不一样。

## 12.4. 一切围绕生产

![](/images/dongge-apple/12/12-05.webp)

当然，普通人难以奢望获得 “诺贝尔奖” 那么大的成就。然而，人人皆可生产，却又是不争的事实。只要你是生产者，或者你是创造者，在人工智能这把宝剑面前，你就是亚瑟王。

我认为，在这个时代，对每一个人来说，最基础且最重要的观念，只要一个词的话，就是 “生产”，两个词的话，就是 “重视生产”，如果是一句话的话，就是 “一切围绕生产”，无论是学习、工作还是生活。仅因为 “生产是唯一正当的财富来源”。

## 12.5. 一切都是语文课

![](/images/dongge-apple/12/12-08.webp)

已经不知道反复提到过多少次了：

> 剩下的，只不过是**中小学生水平的阅读理解能力**……

也许，“语文” 是学校里设置密度最高，却又最不被重视的科目 —— 因为没有人认为自己 “母语” 不过关…… 过去，这好像也没什么。突然之间，因为人工智能的出现，“不重视语文” 成了一个问题，大问题，极其严重的问题 —— 因为人工智能可以直接使用自然语言了。

其实，人类的高速发展，建立在文字基础之上。没有文字，人类就只能像其它动物那样，靠遗传和变异在这个世界里缓慢且又随机地发展。有了文字之后，人类不仅可以即时沟通，还可以跨区域、跨时代地沟通，并且还能进行越来越复杂的记录、传播、思考、沟通……

毫不夸张地讲，一切的文明，都建立在文字之上。

每一次书写工具的改良，每一次文字的标准化，每一次传播工具的变革，都能同时引发整个世界巨大的进步。擅用文字的人越多，文字的传播就越多、思想的碰撞就越多，创新就越多…… 根据梅特卡夫定律，一个网络的价值与该网络中的节点数量的平方成正比。这就是人类的发展曲线越来越陡峭的最根本原因。

现在的人类不仅拥有语言文字，对其还有更为精确的划分：灵活但有些含混的**自然语言**，以及简单精确且毫无歧义的**人工语言**，在科学技术领域，比如，数学、物理、化学，逻辑、编程，人们所使用的都是人工语言。

曾经，有些人理解有误，非得把知识分为文科理科，好像它们是截然不同的两类东西，甚至有可能相互对立一样。然而对大脑来说，一切都是语文课，若是非说有什么差异的话，无非是一个可能使用自然语言更多，另外一个使用人工语言更多 —— 可问题在于，完全只用自然语言就完不成精巧复杂的工作，完全使用人工语言就无法更大面积传播。

所以，无论学什么知识，学什么技能，对大脑来说，其实都是语文课 —— 哪怕那些看起来完全属于体力范畴的技能，本质上也都是如此…… 大脑要理解，要总结，要归纳，要复习，要检索，要调用，要反思，更重要的是，总是要沟通 —— 这些都无一例外地依赖语言。

很多家长根本没反应过来，压根就不是什么 “我家孩子没有数学天赋”…… 99% 的情况下，数学不好是因为语文没学明白 —— 阅读理解能力差。题目其实并不难，关键在于，题目白纸黑字写在那里，却没有读全，没有读懂，漏了个条件，或者没想到另外一个可能性…… 所以题目才会做错。

你得成为一个 “重视语文” 的人，无论是母语，还是外语。你若是父母，你也得影响自家孩子，高度重视语言 —— 因为 “阅读理解能力” 是一切的 “基础技能”，没有它，什么都学不好，甚至什么都学不了，更别谈什么 “生产” 或者 “创造” 了。

然而，它就是相当地耗时费力。

母语的语文，在中小学 12 年天天都可能有语文课，很多人并未真正过关。熟练的英文阅读能力，是很多人后面再补 4 年本科也实际上并未能够掌握的。现在，有了 AI 的辅助，原本需要十数年的积累，即便这个时间就算缩短，通常也不会少于两三年。只能靠耐心，没有耐心，就…… 请自行填空。

## 12.6. 调整大脑的安全阈值

![](/images/dongge-apple/12/12-09.webp)

关于 “为什么小朋友学得更快更好”，有很多理论，也有很多科学研究。其中最有说服力的，用我的话讲，就是：

> 对小朋友来说，那不是在学，那是在 “**为了生存而挣扎**”……

我们的大脑永远痴迷于节能，我们的大脑也最怕死 —— 之所以如此，就是大脑要保障整个身体的安全，毕竟，大脑是必须被其它器官支撑才有意义的器官。

所以，大脑绝对不会允许你超负荷工作。正如汽车发动机若是发挥其 100% 效能的话，它工作不了多久就有可能报废。所以，工程师会为发动机设定一个安全阈值，比如，能启用的最大功率是实际最高功率的 70%…… 这样才能保证安全，还能保持长期稳定。

小朋友为什么 “什么都学” 呢？因为在他们的视角中，所有的一切，都是 “生存” 必须的东西 —— 既然大人们都能做到，我也必须做到…… 就这么简单。那时候，他们的大脑，对任何事情所设定的阈值都相对较高，恨不得总是 100%。

为什么所有的人到最后都不再像小时候那么主动，那么好动，无论对什么都跃跃而试了呢？因为他们的大脑已经反复确认过，“那东西就算不会，也没啥，该吃吃，该睡睡，又不是活不下去……” 所以，大脑会把安全阈值降低，不再是 90% 或者 85%，而是 60% 或 50%，甚至更低。

很多的时候，学没学会，练没练好，跟一个人的智商没有因果关系。如果非说有 “因果关系” 的话，那么，与很多人以为的相反，智商不是 “因” 而是 “果”。因为大脑给自己设定的阈值低，所以，半途而废，造成的结果是，“智商” 并未提高，或者反过来说，“相对变低”。

为什么有那么多的人，读了很多书，听了很多道理，却依然毫无行动，毫无变化？和他们的大脑 “暗中” 给自己设定的安全阈值有关系 —— 他们自己并不知道而已。也因为他们自己并不知道，所以才很焦虑，经常躲到某个角落，抓着自己的头发，发出 “天问” —— “到底哪儿出了问题呢？！”

通过几十万年的基因遗传积累下来的进化成果并不适应这个知识爆炸的时代。大脑内部是一个有神经元构成的比银河系还要复杂不知道多少倍的网络，知识是更为无垠且不断扩张的网络。整个世界随时都在发生着变化，趋势一直就是变化越来越快，变化越来越大。说实话，大脑很难适应这些变化，更不适应这样的变化速度。

最不适应的，可能就是那些大脑默认的安全阈值。过去的几十万年里，生存一直是问题，所以，大脑要不顾一切地先保障生存。突然之间，生存不是问题了，生活环境超级安全，生活必需的成本越来越低…… 

在这种情况下，遵循一切大脑以几十万年的经验而设定的安全阈值，那就只能是所谓的彻底躺平，什么都不用干才对。一日两三餐，每餐一杯牛奶两个鸡蛋半个苹果就已经足够生存。生存已经不是问题了，并且应该一直不是问题，那还不躺平？

现在你明白了，所谓的 “躺平”，从脑科学角度出发，可以有清楚的定义：

> 把大脑的安全阈值降到最低……

所以，所谓的 “积极”，就是反过来而已：

> 想办法拉高自己大脑的安全阈值……

送你一个词，仅仅四个字 —— 就能在必要的时候把你那大脑的安全阈值有效拉升：

> **生不如死**

用它造句的模版主要有三个：

> * 造不出……，生不如死。
> * 练不好……，生不如死。
> * 学不会……，生不如死。

看似简单，却非常奏效 —— 因为我们的大脑真的怕死。大脑这个东西很好玩，谁都欺骗你的大脑，尤其是来自你自己的欺骗，最直接，最有效，往往还是 “即时奏效”。

必要的时候，就用这些模版造句，时不时对自己说几遍，一两天下来，你的大脑就会对此刻骨铭心，对此深信不疑，而后暗戳戳地把那个 “安全阈值” 拉高…… 自然而然地，你更有动力，更有耐力，竟然还不觉得累，也不觉得烦，甚至一切都变得很有趣 —— 这种神奇，体会过一次，就永远放不下。
